Phase-Specific Observation System
Global Rule

When tutor selects a topic, the system already knows:

Current Topic
Current Phase
Previous Stability

Then Topic Management opens the correct observation form for that state.

Not a generic form.

Important Clarification
Stability should influence:
scoring thresholds
interpretation sensitivity
transition strictness

But the observation categories should be driven mainly by phase.

Why?

Because phase tells you what matters most right now.

Stability then changes:

how strict the system is
whether a result means reinforce / remain / progress / regress

So the clean structure is:

Phase determines what tutor observes
Stability determines how the system interprets it

That is the right architecture.

The Full Structure

For every active topic state, Topic Management should show:

1. Phase-Specific Observation Block

What tutor must observe now

2. Phase-Specific Scoring Logic

How those observations are weighted

3. Stability-Aware Interpretation Logic

What the system concludes based on previous stability

4. Fixed Auto Outputs
new score
updated stability
remain / progress / regress
next action
summary

What stability changes actually mean

This is important.

Low

Breaks frequently in the current phase

Medium

Sometimes stable, sometimes unstable

High

Mostly stable in the current phase and ready for next phase if score conditions are met

So the tutor does not “decide” stability.

They only select observations.

The system uses:

current phase
previous stability
phase-specific score
to update the state.
