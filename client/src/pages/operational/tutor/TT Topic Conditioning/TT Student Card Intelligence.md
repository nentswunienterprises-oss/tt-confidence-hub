TT Student Card Intelligence
Active Training State

This is the tutor’s control panel for one student after:

parent subscribes
proposal is accepted
active training begins

The student card is not a profile.

It is:

a live operating surface for Topic Conditioning

Everything on it must answer 4 questions instantly:

What topics are active?
Where is each topic in the OS?
What do I do next in each topic?
What has already happened and what does it mean?
Core Design Principle

The tutor should never have to ask:

What should I focus on?
What phase is this topic in?
What do I do next?
What should I log?
How do I explain this to the parent?

The card should answer all of that.

1. Topic Conditioning
The operating core

This is where TT lives.

Tab A -Map

This is the clarity engine.

Purpose:

Show exactly where the student is in each active topic, what it means, and what direction the tutor must move.

This tab is not for logging.

It is for:

understanding
direction
next action
Map Layout

Each active topic appears as its own card.

Example Topic Card

Algebraic Expressions

Current Phase: Structured Execution
Current Stability: Medium
Status: In progress
Next Move: Increase independent problem volume
Constraint: No time pressure yet

Then beneath:

Phase Track
Clarity → Structured Execution → Controlled Discomfort → Time Pressure Stability
✓           ACTIVE                 LOCKED                    LOCKED

Then beneath:

Current State Meaning

Tutor meaning:
Student understands the topic but does not yet execute it consistently without guidance.

Parent meaning:
Your child is becoming more consistent in solving problems, but still needs reinforcement to apply the method independently.

Then beneath:

Direction

System direction:
Remain in Structured Execution. Reinforce structure every time. Boss Battles may be introduced lightly. Do not move to time pressure.

What the Map must show for every topic
1. Topic Name

Pulled from:

parent enrollment
tutor-added active school topics
ongoing school relevance
2. Current Phase

One of:

Clarity
Structured Execution
Controlled Discomfort
Time Pressure Stability
3. Current Stability

One of:

Low
Medium
High
4. Compressed State Summary

This is the TT Next Action Engine in visible form.

Example:

Clarity + Low

Tutor view: Teach
Parent view: Building foundation
Constraint: No Boss Battles, no time pressure

Execution + Medium

Tutor view: Repeat and reinforce
Parent view: Becoming more consistent
Constraint: Do not rush to time pressure

This is where all engines align.

5. Next Action

Direct tutor move from the Next Action Engine.

Not suggestions.
Fixed instructions.

Example:

Reinforce Vocabulary, Method, Reason

Introduce Boss Battles carefully
Increase timed repetitions
6. Rules / Constraints

This is critical.

The tutor should also see what is not allowed in the current state.

Examples:

No time pressure
No rescuing
Boss Battles not primary
Do not stay in teaching mode
Do not increase difficulty aggressively

This prevents drift.

Tab B -Topic Management

This is where the tutor logs what happened.

Purpose:

Surgically record what was touched in the session and let the system update topic state.

This is not a notes page.

This is a structured logging engine.

Topic Management Layout

Tutor selects the topics touched in the session.

Because one session may touch:

one core topic
one secondary topic
one carry-over topic

But each topic must be logged separately.

For each touched topic, tutor logs:
Section 1 -Session Context

Auto-filled:

Topic
Current Phase
Previous Stability

Tutor cannot edit these.

Section 2 -Exact Tutor Observations

Selections only.

A. Start Behavior
Immediate start
Delayed start
Needed prompting
Avoided / refused
B. Execution Integrity
All steps followed
Minor step skips
Frequent step skipping
Guessing instead of method
C. 3-Layer Breakdown
Vocabulary confusion
Method breakdown
Reason unclear
No major conceptual error
D. Response to Difficulty
Not tested
Stayed composed
Hesitated but continued
Froze
Rushed
Asked for help early
E. Time Behavior
Not tested
Maintained structure
Slight rush
Lost structure
F. Intervention Used
Re-modeled concept
Guided first step
Reinforced method
Introduced Boss Battle
Ran timed attempt

That’s it.

No essays.

Then the system does the rest

After tutor selections, system automatically calculates:

1. Session Score

Response Integrity Score

2. Updated Stability

Using rolling logic

3. Phase Decision
remain
advance
regress
4. Next Action

From Next Action Engine

5. Tutor Explanation

Short internal explanation

6. Parent-facing meaning

Used later in reports and parent UI

Topic Management Must Also Show

Before the tutor submits, the right side or bottom should display:

Live Interpretation Preview

Example:

System Interpretation:
Student remains inconsistent in structured execution for Algebraic Expressions.

Decision:
Remain in Structured Execution

Next Move:
Increase independent problem volume and reinforce step consistency.

Constraint:
Do not introduce time pressure.

This is huge.

Because now the tutor sees:

what they selected
what it means
what the system will do

That trains tutors while they log.

Tab A -Reports

This is not raw logs.

This is:

auto-interpreted individual session summaries
weekly reports
monthly reports

Everything generated from Topic Management logs.

Reports Layout
Section 1 -Session Logs

Auto-generated, per topic touched

Example:
Algebraic Expressions -Session Log
The student is improving in structured execution but still shows inconsistency in step completion. Independent starts are increasing, but reinforcement is still required before progression.

This comes from:

session selections
score
phase
stability

Not tutor writing.

Section 2 -Weekly Report

Organized by topic.

For each active topic, show:

current phase
current stability
movement this week
dominant breakdown pattern
next action

Example:
Fractions
Current Phase: Controlled Discomfort
Stability: Medium
This Week: Student showed less freezing when problems became less familiar. Hesitation remains, but first-step initiation has improved.
Next Focus: Increase Boss Battle frequency without rescue.

Section 3 -Monthly Report

Higher-level output.

For each topic:

start phase
current phase
stability trend
behavioral shifts
next month focus

Then overall:

global response trend
strongest conditioned topic
weakest active topic
transfer readiness

This feeds:

parent UI
leadership oversight
retention confidence
