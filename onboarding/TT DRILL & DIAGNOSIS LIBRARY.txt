🔷 PHASE 1 -CLARITY (REFERENCE ONLY)

LARITY (REFERENCE SPEC)
PURPOSE

Can the student see the problem clearly before solving?

Clarity is NOT solving.

Clarity =

naming what’s there
recognizing the method
understanding why

If this fails → everything else collapses.

1. PHASE STRUCTURE
Core Components (Fixed)

These are the only things that matter in Clarity:

Vocabulary (terms)
Method (steps)
Reason (why it works)
Immediate Apply (can they repeat after seeing)

Each field maps to:

0–44 → regress to LOW

Question:

Is clarity stable enough to move forward?

0–49 → regress to MEDIUM

9. TRAINING DRILL STRUCTURE (REFERENCE)
SESSION = 3 SETS (9 REPS)
SET 1 -MODELING (2 reps)

Purpose:
Build mental map

System behavior:

tutor models (Vocab → Method → Reason)
student explains back
SET 2 -IDENTIFICATION (4 reps)

Purpose:
Recognition without solving

Student must:

name terms
identify type
state steps
explain why
SET 3 -LIGHT APPLY (3 reps)

Purpose:
Test clarity during solving

Student:

solves
minimal guidance

Student does NOT solve

vocab
type
steps

Tests:

start
structure
clarity carryover
11. MASTER RULES
Progression Rule

Only move forward when:
Clarity = HIGH and stable

Regression Rule

If score drops:
→ move back one stability level

System Rule

Tutor does NOT decide next step
System outputs next action

12. WHAT THIS MEANS (REALITY)

Clarity is not “understanding”.

Clarity is:

Recognition + articulation + repeatability

If a student:

can’t name it
can’t describe steps
can’t explain why

They are still in Clarity.

No matter how many answers they get right.
