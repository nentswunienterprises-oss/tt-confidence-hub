TT DRILL SYSTEM FINAL FORM
Core Rule

Every session runs ONE fixed drill per topic-state.

No variation.
No improvisation.

The only variable:

the numbers/content inside the problems

Everything else is fixed.

What a Drill Is

A drill is:

a structured set of reps designed to test one phase-state

Each drill contains:

Fixed number of reps
Fixed difficulty progression
Fixed order
Fixed observation checkpoints
Fixed logging triggers
Tutor Role

The tutor does NOT:

design session flow
decide how many problems
decide difficulty progression

The tutor ONLY:

prepares problems that match the drill structure
runs the drill
logs observations per rep
System Role

The system:

defines the drill
defines what each rep is testing
defines what must be logged
calculates score + stability
GLOBAL SESSION STRUCTURE

For a topic:

1 Drill = 1 Session Block = Full Observation Cycle

No partial drills.

Always complete.

DRILL DESIGN PRINCIPLE

Each drill must:

Start where the student currently is
Push one level forward
Observe breakdown
Confirm stability or instability
