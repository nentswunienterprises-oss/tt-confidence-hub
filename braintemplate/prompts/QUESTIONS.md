# Prompts Questions

How to instruct the system to execute work.

## Execution

- **What is the immediate, approved goal or task?**
- **What is the title or brief description of this goal?**
- **Which files or directories are in scope for changes?**
- **What is explicitly out of scope for this task?**
- **What requirements must the implementation satisfy?**
  - Code style or convention requirements?
  - Testing requirements?
  - Security or secret-handling requirements?
  - Commit and documentation requirements?
- **What are the deliverables for each step?**
- **How should code changes be verified (tests, linter, manual checks)?**
- **What commands or processes should be run before submitting?**
- **How should the implementation plan be structured or reviewed?**
- **Are there any blocking dependencies or prerequisites?**

## Strategy

- **What is the long-term vision for this system?**
- **What are the core principles or values that guide development?**
- **What are the key pillars or foundations the system is built on?**
- **Who are the primary users or stakeholders?**
- **What problem does this system solve, and for whom?**
- **How should the system scale or evolve over time?**
- **What is the business model or revenue strategy?**
- **What is the competitive advantage or unique positioning?**
- **How should the system communicate its value externally?**
- **What metrics or outcomes define success?**
- **Are there institutional or ethical principles that are non-negotiable?**
